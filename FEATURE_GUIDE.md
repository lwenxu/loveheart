# 爱心页面新功能说明

## 功能概述
在爱心动画页面（index.html）停留30秒后，会触发一个互动环节，要求用户通过摄像头做出特定手势来解锁回忆模式。

## 功能流程

### 1. 触发时机
- 用户进入爱心页面后，30秒后自动显示提示信息
- 提示文字："请双手越过头顶比爱心 - 解锁我们的回忆"

### 2. 手势检测
- **要求动作**: 双手越过头顶，比出爱心手势
- **检测方式**: 使用 MediaPipe Hands 进行实时手势识别
- **检测条件**:
  - 必须使用双手
  - 双手需要在画面上部（模拟越过头顶）
  - 两手的拇指和食指需要靠近，形成爱心形状
- **检测进度**: 需要持续保持手势一定帧数（20帧）才能成功

### 3. 用户界面
- **摄像头预览**: 右上角显示320x240的摄像头画面
- **手部追踪**: 实时显示手部关键点和连线
- **状态提示**: 摄像头下方显示当前检测状态和进度

### 4. 回忆模式
成功检测后进入回忆模式：
- **图片播放**: 自动播放 memorys 文件夹中的90张照片
- **文字展示**: 每张图片配有感人的文字描述
- **切换效果**: 图片和文字都有淡入淡出的过渡效果
- **播放时长**: 每张图片显示4秒
- **结束语**: 所有图片播放完后显示 "这就是我们的故事 💗"

## 技术实现

### 使用的库
- **MediaPipe Hands**: Google的手部追踪模型
- **MediaPipe Camera Utils**: 摄像头工具库

### 核心功能
1. **手势识别算法**: 检测双手的拇指尖和食指尖距离，以及手腕在画面中的位置
2. **图片轮播**: 使用定时器控制图片切换，配合CSS过渡效果
3. **状态管理**: 防止重复触发，确保流畅的用户体验

### 文字内容
准备了90条感人的文字描述，循环匹配图片，内容包括：
- 相遇的回忆
- 相处的美好时光
- 对未来的期许
- 爱的表白

## 用户体验亮点
1. **互动性强**: 需要用户主动参与手势识别
2. **情感深厚**: 精心准备的文字配合珍贵的照片
3. **视觉效果好**: 淡入淡出的过渡效果让切换更加柔和
4. **仪式感足**: 从爱心动画到手势解锁再到回忆播放，层层递进

## 注意事项
- 需要用户授权摄像头权限
- 建议在光线充足的环境下使用，以提高手势识别准确率
- 手势需要越过头顶（在画面上部），不是普通的比心手势
